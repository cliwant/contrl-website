What Happens as LLMs Improve?
Problems LLMs will solve vs. Problems only contrl can solve
What LLM Advances Will Solve
= Areas replaceable by pure LLM capabilities.
Text Generation Quality
Writing quality, logical structure, context understanding. Auto-improves as models get better.
contrl impact: Low — Planner prompt quality keeps pace
Long Document Processing (Long Context)
Understanding 100p+ RFPs in one pass. Context window expansion increases reference file accuracy.
contrl impact: Positive — Planner input quality auto-improves
Multilingual · Domain Terminology
Industry-specific terminology, mixed-language usage, natural business writing style.
contrl impact: Positive — Content analysis accuracy improves
Generic Document Generation (from scratch)
Ask ChatGPT/Claude to "make a proposal" and it can generate text drafts and even .pptx via python-pptx.
Threat: High — New doc generation is a competitive area. Existing template DNA preservation is our differentiator.

What LLMs Won't Solve
= True moat + temporarily valid advantages + positioning.

DNA Log — Company Knowledge That Evolves Over Time
Company DNA is not static. Track records are added, capabilities change, tone evolves.
DNA Log tracks these changes over time, enabling any point-in-time DNA snapshot to be reproduced.

The Flywheel DNA Log Creates

Point-in-time Reproduction — Requests like "company intro as of March 2026" use the DNA snapshot from that date
Change Detection — If recent documents show a tone shift, automatically suggests tone profile update
Switching Cost — After 6 months of accumulated DNA, moving to another tool means "starting over" — this is the moat
